# 更新日志

## v0.2.4 (2026-04-16)

### 改进

- **macOS 置顶更稳定** — 置顶窗口改用真正适合主窗口的 Floating 层，不再误用弹出菜单层级
- **跨桌面和全屏场景更稳** — 置顶时会同步加入 `CanJoinAllSpaces` 和 `FullScreenAuxiliary`
- **切换窗口后继续保持在前** — 置顶时会主动重新前置，减少被其他普通窗口盖住的情况

---

## v0.2.3 (2026-04-16)

### 改进

- **拾取窗口更直接** — 在目标窗口上按下鼠标键就会立即确认并停止拾取，快速点击也不会漏掉
- **macOS 权限识别修正** — 已经允许发送键盘事件时，界面不再继续误报“需要辅助功能权限”
- **macOS 置顶真正生效** — 标题栏📌按钮现在会真实切到置顶层，不再只是保存状态

---

## v0.2.2 (2026-04-16)

### 新功能

- **默认中文** — 首次启动默认使用中文界面，不再默认 English
- **macOS 窗口拾取** — 现在可以在 macOS 下选中鼠标所在窗口
- **macOS 按键发送** — 选中目标后可以把按键发到对应应用进程

### 改进

- **权限提示更直接** — 缺少辅助功能权限时，界面会直接提示原因并提供“打开设置”按钮
- **Start 状态更清楚** — 没权限或没选窗口时按钮会禁用，并在下方直接显示不能启动的原因

## v0.2.0 (2026-04-15)

### 新功能

- **全局热键 F9** — 无需切换窗口，随时按 F9 切换启动/停止
- **窗口置顶** — 标题栏📌按钮，点击后窗口始终在最前面
- **配置持久化** — 按键任务、发送模式、置顶状态自动保存为 JSON，下次启动恢复
- **实时统计刷新** — 状态栏计数每 500ms 自动更新，无需手动刷新

### 改进

- **SendMode 简化** — 仅保留 PostMessage 和 SendMessage 两种后台模式
- **PostMessage 修复** — lparam 包含正确的扫描码（MapVirtualKeyW），发送 WM_KEYDOWN→WM_CHAR→WM_KEYUP 三连
- **SendMessage 模式** — 同步发送方式，兼容更多应用
- **状态栏热键提示** — 底部显示 "F9: Toggle" 提醒

## v0.1.0 (2026-04-15)

### 新功能

- **窗口拾取** — 点击 Pick 按钮后移动鼠标，实时显示光标下的窗口名称，再次点击锁定目标窗口
- **多按键独立调度** — 支持添加多个按键任务，每个按键独立的时间间隔，互不干扰
- **按键录制** — 点击按键徽章进入录制模式（橙色闪烁），按下任意按键即完成切换
- **间隔编辑** — 每行的时间间隔为可编辑输入框，直接输入毫秒值
- **后台按键发送** — 使用 Windows PostMessage API，目标窗口无需置顶即可接收按键
- **启停控制** — Start/Stop 按钮控制所有任务的统一启停
- **状态栏** — 底部实时显示运行状态和每个按键的发送计数
- **自定义无边框窗口** — 暗色主题 UI，支持标题栏拖拽移动、最小化、关闭按钮
- **默认配置** — Enter 键，200ms 间隔

### 技术栈

- Rust + GPUI (Zed 编辑器的 GPU 加速 UI 框架)
- gpui-component (UI 组件库)
- Windows API (PostMessage / WindowFromPoint / GetAncestor)
